1 Introduction

1.1 Installing Ruby

Before you can start you must check if you have installed Ruby. To do this you can type in a console: ruby --version. The output can be similar to that one:

ruby 2.2.1p85 (2015-02-26 revision 49769) [x86_64-linux]

.

If there are no ruby inside the output you have to follow the next section. Otherwise you can skip that step and go to Section 1.2, “Installing publican_creators”.

1.2.1 Running the setup #

For running the setup go to the gem's directory with cd .rvm/gems/ruby-2.4.1/gems/publican_creators-1.2.0. Then just type rake setup_start.

Important

Keep in mind, that the 2.4.1 from ruby-2.4.1 must match your chosen version.

The setup script does the following:

  • It checks on which host os (Linux) it runs. Then it reacts to that and tried to install the hard dependencies yad and publican.

    publican_creators needs a 4.x version of publican because it uses the parameter “--dtdver”. In case of a Ubuntu host os the setup adds a Launchpad PPA which provides backported versions of publican 4.x for Trusty (14.04) and Utopic (14.10). Vivid Velvet ships a usable publican 4.x version. For other Distributions you have to check if a 4.x version is available for your os. Otherwise publican_creators will NOT work.

  • If you have already a .publicancreators.cfg in your Homedirectory it will add the actual shipped .publicancreators.cfg as .publicancreators.cfg.new in your Homedirectory. You have to check, if new parameters are added or depreached one are there. So modify your old .publicancreators to match the new one.

  • The last thing in the setup is to install two *.desktop files for PublicanCreators and RevisionCreator in .local/share/applications. This means it will shown in the program launcher.

  • If you aren't have a old .publicancreators.cfg the setup copies a actual version in your Homedirectory and will launch a Texteditor with the config loaded. So you can modify it to your needs. The config file will be explained in the next chapter.
